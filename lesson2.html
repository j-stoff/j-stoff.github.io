Lesson 2 - Variables

Taking a look at the HelloWorld.java in a text editor, some of the words should seem familiar. For example, public and class are certainly easy to understand but have a very specific meaning in Java. Such words are called keywords. Keywords can not be used as names for variables, methods, or classes as these words are used by the compiler to indicate different things during compilation.


public class HelloWorld {}

The above is a valid class declaration (though not very useful)

In it, the code is saying I have a class called 'HelloWorld' that is for public use.


public static void main(String args[]) {}

Above is the main method for almost every java program, something the JVM looks for in order to excute code.


System.out.println();

Analyzing the last portion, the meaning is that there is a System Object that Java knows about, it has a field called out for standard output (the .out), and a method called 'println' which can take almost any arguement and print it to the console with a carriage return and new line feed.

This is one of the most important debugging tools to come across, though make sure to take it out when finalizing the code.

Also important to note, Java is based on C meaning each line ends with a semi-colon or curly brace. NO EXCEPTIONS.

After running the HelloWorld program, compiling and running code is that simple.

Note: the file name and the class within the file must match or the compiler will throw an error. This is always true and why version control can be a pain sometimes.

Note: This is also why Java class names are capitalized and use camel case as a standard because the compiler doesn't care if a class begins with a lower case; it just makes sure the file name matches the class name. Convention dictates to capitalize class names in most languages.

Note: There is also a way to compile the .class files into a separate directory to keep the source code separate from executables.

That should have been fairly easy, so onto actually writing some code.

First thing would be to know about Java Data types as there are only two - primitives and pointers to objects. The first, primitives, are classically what you might think of as variables, integers, characters, boolean values are all primitives. The other, object references (pointers as I have been calling them), represent the vast majority of variable types used. Even String is an object, not a primitive.

A full list of primitives is below:

  • boolean - true or false
  • int - from -2^31 to 2^31 - 1
  • short - -32,768 to 32,768
  • char - 16 bit unicode value. use single quotes.
  • float - 32 bit decimal number (use double most of the time)
  • double - 64 bit, decimal number
  • long - 64 bit, -2^63 to 2^63 - 1
  • byte - 8 bits, from -128 to 127

Everything else is a refence (pointer). Even Strings.

Onto another topic: Operators

Operators are present in all languages I know of though sometimes in different forms. They are very similar to mathematical operators with some variations.

The list of common operators in Java:

  • =   is the operator for assignment
  • +   is for mathematical addition or string concatenation
  • -   is for subtraction
  • *   is for multiplication
  • /   is for division
  • %   is the modulus which returns the remainder
  • .   is for accessing the object

The order of operation is at a default from left to right. Multiplication and division have a higher precedence than addition and subtraction. Assignment is one of the lowest predences.

Note: there are other operators, such as ++ or ?

Additionally, some of these operators can be combined, such as += which will add first the value on the right to the variable then assign it to itself.

A more explicit definition of above would be: x = x + 1, is the same as x += 1

Logical Operators

The logical operators return true if the evaluated statement is true or false if it is not.

The standard greater than, greater than or equal to, less than, less than or equal to are present.

For example (3 > 2) would return true. While (3 < 2) would be false.

Additionally, there is also >= for greater than or equal to, and <= for less than or equal to.

Judging equality is with ==

Note, all of these ONLY WORK FOR PRIMITIVES. Comparing objects is done with the .equals() method.

Declaring variables

Java is a strongly typed language meaning that the 'type' of the variable must be declared when the variable is declared.

For example:

 int index;

Simply declares an integer called index for further use.

Other examples:

 double bigNumber;
 boolean didItWork;

Note for chars: Chars must use single quotes when declaring a single character. Double quotes indicates a string. The converse isn't always true as 'this is a string' is a string.

Next, assignment of the variable to an initial value can take place in one step

 int index = 10;

The index value starts at 10 instead of 0 (that is where integers begin).

It is a good idea to explicitly initialize variables in order to avoid silly mistakes.

Side note, when using numbers, typically only use integers and doubles. Floats, longs, and shorts have specific use cases for memory preformance but the difference between a 32 bit float versus a 64 bit double is not much. Changing a bunch of code from a float to a double would be.

Strings

Strings are Object references in Java but are quite unique. In many ways Strings are treated like primitives in Java but they are not.

Take a look at how to assign a String below.


String name = "Bob";

This creates a new String Object with the value "Bob" in it.

String Concatenation

String concatenation is the combination of two String Objects into one. For example, say you wanted to concatenate your fist and last names together.


String firstName = "Bob";
String lastName = "Goodname";
String fullName = firstName + lastName;

The value in the fullName variable is "BobGoodname" as the Strings have been mashed together. Note that the result isn't very readable to humans which is why use spaces, tabs, or commas to separate string values before outputting them.
