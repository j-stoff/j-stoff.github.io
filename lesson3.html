Lesson 3

More than just variables

Obviously programming is much more than just declaring variables and assigning them to values.

That is where decisions are made in the program, using if statments or even switch statements.

Additionally, there is looping to do repeatious tasks using for and while

The big IF

An if statement has a fairly simple structure.


                    if (statement to evaluate) { 
                        //code to run goes in here
                    }

Within the parenthesis, anything that evaluates to a true or false statement can be put there. Even assignment returns a true or false!

Additionally, an else can be added after the if statement to execute when the if statement is false.

The chaining of if statements is also possible with else if statements.

Code example


                    if (true statement) {
                        // if true stuff
                    } else if (different statement) {
                        // else if true stuff
                    } else {
                        // Else
                    }

For is For and when to while

Put simply, for and while loops accomplish the exact same task but have different use cases in my opinion.

Typically, I use for loops when the number of loops is predetermined. For example, looping through an array or list, there is usually a field to indicate the size of such a data structure. Meaning that the maximum number of loops should be equal to 1 less than that size.

While is used when the number of loops isn't known, until some condition being evaluated in the while turns false.

Additionally, while loops have the added step of declaring variables outside of the loop or they will be reinitialized each time.

While Loops

The structure of the while loop is similar to the if statement except that it loops around.


                        while (condition to evaluate) {
                            //Code to run
                        }

As mentioned above, if variables are used to evaluate the loop condition they must be declared outside of the loop or they will be re-initialized each loop.

For Loops

For loop structure is more complicated. For loops can initialize values within the parenthesis. Semi-colons serve as seperators for the different declarations.


                        for (initialization value; evaluation statement; increment/change variables) {
                            // Code to loop
                        }

Example: for(int i = 0; i < 100; i += 1) declares a variable i (horrible name by the way) which loops until it reachs 100 or greater, incrementing by 1 each loop

Switch statement

Certainly more powerful than simple if statements and is meant to replace long if/else if chains that become unmanagable.

Overall structure is quite different from others


                        switch (condition) {
                            case (expected value without parenthesis):
                                // Code to run
                                break;
                            case (Another expected value):
                                // Code
                                break;
                            default:
                                // Usually want to have a default in case of fall through
                                break;
                        }

Note be sure to break after each case, or all cases will be evaluated.

Caution with loops and switch statements

Both looping and switch statements can lead to infinite loops if the conditions never change.

A break statement will get out of the current loop or switch case statement.

Loops also have a continue statement which will skip the rest of the code in the loop and start again at the top of the loop.
