Some terminology

Object

An object is simply a container for variables and methods. Every class inherits from the Object type (more on inheritance later).

Class

A class is a type of object but Java recognizes code at the class level, not at the object level. Almost every bit of code in Java is within a class. Similar to how C requires a 'int main' function to run the code, Java uses classes as the backbone to run code.

Method

A method is a function within a class. Typically this gives the class some functionality, so for example, a BufferedReader Class contains methods that would be useful for reading a file's contents such as readLine() which reads one line of the file and advances file pointer.

Instance variable

Similar to 'properties' in other languages, they comprise local variables within a class that are required for a class to operate. Going on with the BufferedReader class example, a useful instance variable would be a String containing the name of the file.

Compiling and running the code

At the command line, type 'javac Hello' and press tab to auto-complete the HelloWorld.java file. This will compile the code into HelloWorld.class file.

Run the code with 'java Hello' and again tab to complete the proper sequence. Note you do not need to add 'HelloWorld.class' as the computer will know that 'HelloWorld' is executable.

Note at the console in the next line, "Hello World!" should be printed as the program ran successfully or a compilation error happened.
