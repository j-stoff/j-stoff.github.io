Lesson Six

Scope

By now, the 'public' keyword has been used quite a bit with no explanation.

'public' is known as an Access Modifier as it declares how the information can be accessed.

Public

Public is the simplest example. Anyone and anything can access public variables and methods.

Protected

Protected indicates that only classes that inherit from the class hierarchy can access these methods and variables.

Private

Private is the strictest as only the class has access to the these variables and methods.

Default

The basic privelege given if no access modifier is given. Not as strict as protected but not as open as public.

Note: All instance variables should be declared as private. The reasoning is that the variables should not be changed unless through accepted means (see getters/setters)

Having private instance variables increases security as the only way to access the variable is through methods the developer has written.

Getters and Setters

As I said above, the convention for java is that all instance variables of a class are declared as 'private' meaning only the class has access.

That might seem rather limiting as other things might need the information within those variables.

Here is where a get/set methods come into play. Commonly called getter/setters.

The idea behind getters is to return the current value within the variable. A setter assigns the variable to a new value.


private int myAge;

public int getMyAge() {
    return this.myAge;
}

public void setMyAge(int diffAge) {
    myAge = diffAge;
}

Lets analyze the code example above. First an integer is declared called 'myAge'. This is private access meaning only the class knows about it.

Next is the Getter method. It is public and returns an int which is the current value of 'myAge'. Note the use of 'this' keyword. 'this' refers to the class which the code is a part of.

Finally the Setter method. Again it is public but has no return value. Unlike the getter, it has one parameter that will be assigned to the variable.

What is the point?

Couldn't the variables just be default? Or even public.

The answer is yes, but that would be a big security flaw as well as bad pracitice. The reason being, anyone using the code with a little knowledge could change the variables very easily with simple assignment.

Getters and Setters provide a way that if the programmer wants to alter how the variable is accessed or changed, that it is done so in a specific way so the class doesn't screw up.

Constructor

On the subject of skipped topics. The 'new' keyword when declaring a class as well as the reason for parenthesis isn't clear quite yet.

The reason is that the 'new' keyword combined with a class's name and parenthesis calls a constructor.

What is a constructor

A constructor is a method that creates an instance of the class (itself) to return.


public File() {
    // Initialization code
}

Constructors serve to initialize code within a class. For example, if there are some variables that are Objects, they need to be created so they are not null.

Notice above, the constructor has no parameters which is called a No Arg constructor or Empty constructor. It is good practice to put one of these in regardless of other constructors.

But wait, if a class needs a constructor, how did the code run before.

The JVM is smart, it figures out that if a constructor isn't declared that a no-arg constructor is assumed which will work in all cases for the class.

The downside to having a smart JVM is that if any other type of constructor is declared, that is it. That is why it is common to have a No-Arg constructor as well as other constructors that are for convience.

Overloading and Overriding. What are they?

I mentioned above that a class can have several types of constructors. How is that possible? How does the JVM know which constructor to use?

The answer is Overloading

Overloading is when a method or constructor has multiple definitions that have different parameter requirements but preform the same action and return the same type.


// No Arg constructor
public File() {
}

public File(String fileName) {
    // Do something
}

In the example above, the default no-argument constructor is present with no parameters at all. Another constructor takes a String for a given file name.

So what is Overriding?

Overriding is actually very different, but still deals with methods.

Overriding deals with inheriting a method from a class and rewriting it to suit the sub-class.

Recall the Dog example which inherited from Animal class. Animal had an eat() method that had a simple print statement. Well, the Dog class needs to eat() as well but would be exactly like any other Animal class.

Adding an Override would tell the JVM, oh that method I inherited, yeah I want to change that to this definition here.


// Note the @ is an annotation symbol
@Override
public void eat() {
System.out.println("Bark, I like bacon");
}

Note: The @ symbol is for a Java annotation. Annotations can be used for many things and are becoming more popular. Sufficed to say, @ is needed with Override or the JVM has no idea what you are trying to do.
