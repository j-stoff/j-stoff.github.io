Lesson 4

More clarification on previous code

Now we will look at writing our own class definitions. For now, ignore the scope definition (public) for the moment.

As you probably guessed, 'class' in the class definition is create a class of the given name.


                    public class ExampleClass {
                        //Code here
                    }

Onto something new - Variables

How do you add variables to a class?

For now, the same way to declaring as a method works. Though there is a better way, more on that later.

Using the same class as before:


                    public class ExampleClass {
                        int lessonsCompeteled;
                        boolean doingTheAssignments;
                    }

Now the good stuff: Methods

Variables are nice, but they only store things. Useful for later on, but databases are for storage, programs are supposed to do something.

Jumping right into it


                    public void myMethod(String user) {
                        // Code here
                    }

This is a full method declaration. Though the access modifier is not needed, one is used almost always.

The next field is for the return type. It can be a primitive, Object type, or void which returns nothing.

Next is the name for the method. Again, convention has camel case beginning with a smaller case letter and capitalizing new words. The more descriptive the method, the better.

The parentheis are for arguments. When declaraing parameters, the type is needed as well as a local name to be used within the method. In the example above, a String is a parameter with the name 'user'.

Parameters are not optional, but aren't always handled correctly within methods. Be careful what is passed to a method is what it should be.

Lastly, the curly braces contain the code to execute for the method.

Why even put things into classes?

Code reusability. It's that simple. Well written code can be re-used very frequently with little overhead to the developer.

Scope

For now I have been ignoring the 'public' declarations as well as static defitions as they deal with scope.

We will tackle these in the future, but for the moment everything is scoped within the context it was declared.

For example, a variable made in a method will survive the duration of the method. A variable declared within a class is usable as long as the reference to the class is active.


                    public class ExampleClass {
                        int lessonsCompeteled;
                        boolean doingTheAssignments;
                        
                        public void myMethod(String user) {
                            // Some cool code
                            int big;
                        }
                    }

Using the example above. What is the scope for 'doingTheAssignments'? What about 'big' within myMethod?

Creating class refences

The majority of the variables will be references to pre-made classes as they have greater functionality than primitives.

To create a class use the 'new' keyword then the class name with parenthesis. Similar to a function call.


                    ArrayList list = new ArrayList();

The parenthesis actually calls the 'constructor' which is something we will get to soon.

Note: Strings do not require a 'new' keyword for each string.


                    String name = "bob";
